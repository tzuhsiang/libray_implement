專案開發規格書：LibriFlow v1.1.0 (UI/UX 優化版)
1. 修訂目標
提升「個人圖書管理系統」的視覺層次感，簡化書籍評分與狀態更新的操作流程，並增強系統在資料空值時的引導性。

2. UI/UX 變更需求
A. 「新增書籍」區域優化
變更內容： 將原有的固定式大表單改為「摺疊式面板 (Collapsible Panel)」。

規格細節：

預設狀態：僅顯示一個「+ 新增書籍」的按鈕。

點擊後：向下展開表單內容。

送出或取消後：自動收合表單。

技術練習點： Vue 3 的 v-show 條件渲染與簡單的過渡動畫 (Transition)。

B. 狀態標籤 (Status Badges) 色彩化
變更內容： 針對不同閱讀狀態顯示具辨識度的顏色。

規格定義： | 狀態代碼 | 顯示文字 | 建議配色 (Tailwind Class) | | :--- | :--- | :--- | | unread | 未讀 | bg-slate-100 text-slate-600 | | reading | 閱讀中 | bg-blue-100 text-blue-600 | | finished | 已完成 | bg-emerald-100 text-emerald-600 |

C. 互動式星星評分 (Interactive Star Rating)
變更內容： 移除「更新評分」數字輸入框，改為直接點擊星星更新。

規格細節：

滑鼠懸停 (Hover)：星星變色預覽。

點擊 (Click)：立刻發送 API 請求更新後端資料。

視覺回饋：成功後星星維持高亮顏色。

D. 列表空狀態 (Empty State)
變更內容： 當書架無任何書籍時顯示提示。

規格細節：

判斷條件：books.length === 0。

顯示元件：一個插圖 (Icon) + 提示文字「目前書架沒有書籍，開始建立你的第一份書單吧！」。

3. 前端邏輯變更需求
A. 響應式狀態管理 (Ref / Reactive)
需求： 確保書籍清單 (List) 在更新「狀態」或「評分」後，畫面能即時響應。

實作重點： 修改 BookList 內的更新 function，在 API 成功回傳後，直接修改原本 reactive 陣列裡的該筆資料物件，而不是重新拉取整個 API (優化效能)。

B. 防抖處理 (Debounce Logic) —— 選配
需求： 避免使用者快速點擊星星造成連發 API 請求。

規格： 在點擊星星後延遲 500ms 才正式送出 API 請求，若期間有新的點擊則重新計時。

4. 驗收標準 (Acceptance Criteria)
摺疊表單： 進入頁面時不會被巨大的表單佔據視線。
即時更新： 點擊星星後，不需要重新整理頁面，星星顏色就發生變化。
顏色一致性： 「閱讀中」的書籍呈現明顯的藍色標籤。
防錯處理： 若後端 API 斷線，點擊更新後應有錯誤提示 (如：Alert 或 Toast)。
